Game Description

Universe Collide is battlefront-esc game which engages in ship shooter battles between opposing teams or enemies. You play as a vessel any vessel fighting against AI driven opponents until you rein as the ace. You can play as any vessel from the Star Wars and Star Trek franchises. Future inclusions would make vessels from battlestar and some personal custom vessel designs.

Status:

However as a lone developer developing the project, the current version is as a demo version with the crucial features and only as a loose free for all. Additionally, Star Wars is the only franchise that is included and a virtual clone from the original game. However with time, it should develop with more features .....

Project Purpose

The purpose of this game was primarily as a passion project to bring together different vessels into one game. On the other hand, this project also serves as good practice in applying different areas of software design or principles to better construct the game's software architecture.

Approach

As passion project development started early with little design of project management. However during the course of the redevelopment of the projection into Unity's 2019.3.4f engine version, the development has shifted to agile development using the Kanban framework. However no official timeline is decided for the project as it is a continual process of open development and review.

Project Progress

* Progress as of the 4th November 2020

Current progress on the project sits at beyond the level of development intially done before switching to the newer engine. The redevelopment of the project addressed many of the issues seen with the initial development with handling multiple duplicates of values with the flyweight pattern. Additionally in also flipping the process which development took place with the core mechanics and features to be developed before the artistic elements and asset creation. Recently the AI behaviours contained in their own finite states have mostly been converted and are more responsive than their previous iteration. However functionality addressign the UI and the scoring system have yet to be converted before developing other components to extend the concrete implementation.

Current Development Features

  • Completed non stylised main menu with loading screen before heading into main gameplay.
  • Extendable weapon components for quick and modifiable development of custom weapons.
  • Generalised design of base components allowing for inherited instances of enemy AI, friendly AI, player Objects and more.
  • Use of scriptable objects allowing for shared constant values between multiple instances of objects, enabling lightweight designs.
  • Stat handling of core abilities and entitiy stats for centralised read and write of entity's characteristics.
  • Dynamic switching of object states through exchanging back and forth between components managed by the entity's state machine.
