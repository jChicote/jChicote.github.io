Project Description

The AGP assignment is the culminative project on mini projects developed during the Advanced Games Development subject. It undertakes the effort to learn firsthand the use of the Unreal Engine anddevelopment of basic to advanced algorithms used in games. The game produced from this assignment sees the player playing against multiple enemy ai within an offline generated map.

See code in Github

Project Purpose

The project requires ourselves to learn algorithms associated with game artificial intelligence and procedural terrain in Unreal Engine. In games development there are many situations where we will encounter the need to develop some formm of artitificial intelligence in games to allow for dynamic gameplay and interaction. The same goes for terrain generation, since expansive open world games cannot be quickly created , it makes sense for terrain to be generated procedurally for the inclusion of a randomised and dynamic terrain. However, the project was to be developed primarily in C++ as it was necessary for being closely accostomed to the language. Although this came with certain limitations as there was some but yet sparse documentation of different areas of the API, and that Unreal's blueprints were extremely valuable and quicker than whats offered in C++.

Objective

Objectives set up for this subject are very simple and require the developes to develop a coded projects that demonstrated topics or techniques in artificial intellignence and with procedural generation. Additionally goals also included the production of videos that educate/explain and demonstrate the topic or technique created for the project.

Artificial intelligence - A.I Senses (Vision)

To us humans, we operate with our 5 senses to perceive the world around us and the environment we interact with, this itself can also be applied to artificial intellignece and the workings of an agent. The individual contribution to the project involved developed the vision of the AI senses exploring and developing the different zones of vision into determining the type of values that will drive the behaviours used.

Procedural Generation - Environmental Biomes

Procedural generation are an algoriothmically driven and can be used to generate varying biomes vased on the characteristics of the terrain and other known variables. When working with procedural generation often perlin noise was encountered in initially providing the varying landterrain. With this work done on the creating varying biomes relies on height to define the tempurature variation that is used to define the type of plant being spawned. However like in minecraft, there are be other variables that influence the various biomes that can exist. Such variables can include height, tempurature, wind, slope, presence of water, etc. The development for biomes relies on offline generation so that it can be built with the existing land generation algorithms.

A range of test spheres are used in placement of actual biome plants to observe the tempurautre being sipulated at different altitudes, and was used to check whether the generator is working as expected on slopped terrain.
